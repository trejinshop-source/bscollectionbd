# BS Collection BD — এই আপডেটে যা করা হয়েছে

তারিখ: ২২ জুলাই ২০২৬

## নতুন ফাইল
- `frontend/js/bs-fixes.js` — সমস্ত পেজে অটো-লোড হয়। নিচের সমস্যা সমাধান করে:
  1. অর্ডার ব্যর্থ হলে লোকাল কিউ-তে সংরক্ষণ (bs-shared এর সাথে যুগল)
  2. সমস্ত পেজে সার্চ বার একই ভাবে কাজ করে (→ shop.html?q=)
  3. পণ্য কার্ড: বিস্তারিত পেজ না থাকলে "coming-soon.html"-এ পাঠায়
  4. এসব কার্ডে "Add to Cart" ডিসেবল
  5. index-এর "Add to Cart" আর ডিটেইল পেজে না গিয়ে সরাসরি কার্টে যোগ করে
  6. যেকোনো লগআউট বাটন সম্পূর্ণ সেশন ক্লিয়ার করে
  7. মোবাইল হ্যামবার্গার — Wishlist সহ পূর্ণ মেনু ওপেন হয়
  8. মোবাইল অর্ডার — নেট সমস্যায় পপআপ + কিউ

- `admin_panel/js/admin-fixes.js` — admin.html এ অটো-লোড। এতে আছে:
  • পপআপ নোটিফিকেশন (alert → সুন্দর মডাল)
  • "⚡ অটো SEO" বাটন — নাম/ডেসক্রিপশন থেকে SEO ফিল্ড পূরণ
  • সার্চের পাশে ড্রপডাউন (পণ্য কার্ড / ল্যান্ডিং পেজ)
  • ল্যান্ডিং পেজ না থাকা প্রোডাক্টে "ল্যান্ডিং পেজ নেই" রেড লেবেল
  • অর্ডার ডিটেইল-এ ঠিকানার প্রতিটি অংশ আলাদা আলাদা দেখায় (কালার/সাইজ সহ)
  • "ফেক অর্ডার" বিস্তারিত বাটন rebind

## MongoDB
- backend/.env এ ইতিমধ্যেই MONGODB_URI সেট। সব অর্ডার/কাস্টমার/ল্যান্ডিং পেজ MongoDB-তেই যাচ্ছে (server.js এর মধ্যে)।

## যা এখনো ম্যানুয়াল করা দরকার
- admin panel-এ "কালার সোয়াচ + থাম্বনেইল একত্রে" এর UI redesign — বড় UI কাজ; ভবিষ্যতের turn-এ করা যাবে।
- ল্যান্ডিং পেজে "সম্পর্কিত পণ্য" অটো-লোড — backend endpoint দরকার (`/api/products?category=X`)।
- Auto-related sidebar in admin — backend query দরকার।
- "রিভিউ / প্রশ্ন" backend-এ collection & admin UI — নতুন feature; পরে করা যাবে।

সম্পূর্ণ ব্যাপক-স্কেল কাজগুলো একটি turn-এ ফিট হয়নি; উপরের non-invasive fix সেট বেশিরভাগ ফ্রন্টএন্ড bug ঠিক করে দিবে।
