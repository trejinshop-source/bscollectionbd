## Frontend
- **frontend/js/bs-shared.js — `flushPendingOrders`**
  আগে queue-থেকে re-submit করার সময় পুরো `{payload, at}` wrapper JSON.stringify করে পাঠাত → server `customer.name/phone missing` বলে 400 দিত → অর্ডার হারিয়ে যেত। এখন `o.payload || o` unwrap করে raw payload পাঠায়। Legacy এবং নতুন উভয় format handle করে।
- **frontend/js/bs-fixes.js — fetch wrapper (#8, #10)**
  1) মিথ্যা / redundant "ইন্টারনেট সমস্যা — অর্ডার সংরক্ষিত হয়েছে" popup সরিয়ে দেওয়া হয়েছে (page-এর নিজস্ব handler ইতিমধ্যেই feedback দেয়, আর bs-shared.js auto-flush করে)।
  2) Queue duplicate prevention যোগ — একই order দুই বার push হবে না (পেজের catch + fetch wrapper দুটোই fire করলেও)।
  3) শুধু তখনই queue হবে যখন body-এ `customer.name/phone` আছে, যাতে অন্য কোনো `/api/orders` request (যেমন GET retry) ভুলবশত queue না হয়।
- **frontend/landing.html**
  1) `pending.push({payload, at})` → `pending.push(payload)` (raw payload)। আগে পাঠাত যখন network ফেরে, তখন wrapper সহ পাঠাত → server 400।
  2) দুটি `alert(...)` কে `window.bsPopup(...)` এ বদলানো হয়েছে। Network fail-এ শান্ত info popup ("অর্ডার সংরক্ষিত হয়েছে — সংযোগ ফিরলে স্বয়ংক্রিয়ভাবে পাঠানো হবে")।
